11ty (Eleventy) とは？

簡単に言うと...

11ty（イレブンティ）は、「文章を書くだけで、自動的にWebサイトを作ってくれる魔法のツール」です。

普通のメモ帳で文章を書くような感覚で記事を作成すると、11tyが自動的に美しいWebサイトに変換してくれます。HTMLやCSSの知識は必要ありません！

📝 マークダウンで書ける

普通の文章＋簡単な記号でWebページが作れます。例：「# タイトル」→大きな見出しに変換

🔄 自動変換

あなたが書いた文章を、11tyが自動的にプロ仕様のWebサイトに変換してくれます

⚡ 超高速表示

生成されるのは軽量なHTMLファイルなので、どんなサイトよりも高速に表示されます

🎨 自由にデザイン

テンプレートを使って、思い通りのデザインのサイトが作れます

従来の方法 vs 11ty

従来の方法

  • HTML、CSS、JavaScriptを覚える必要がある
  • 1ページ作るのに何時間もかかる
  • デザインの調整が大変
  • 記事を書くたびに複雑な作業

11tyなら

  • 普通の文章を書くだけ
  • 1記事20分で完成
  • デザインは自動で美しく
  • 記事を書けば即座にWebページに

2 11tyプロジェクトの詳細セットアップ
💡 11tyプロジェクトとは？

🤔 初心者向け解説

「11tyプロジェクト」とは、あなたのWebサイトを作るための「材料と設計図をまとめた箱」のようなものです。

この「箱」の中には、記事ファイル、デザインテンプレート、画像、設定ファイルなど、Webサイトに必要なすべてが整理されて入っています。11tyはこの「箱」を見て、自動的に完成したWebサイトを作り出します。

1. プロジェクトフォルダの作成
プロジェクトフォルダを作成（例：my-websiteをあなたのサイト名に変更）
mkdir my-website
フォルダに移動
cd my-website
2. Node.jsプロジェクトの初期化
package.jsonの作成
npm init -y
💡 解説： npm init -y は Node.js プロジェクトの設定ファイル（package.json）を自動作成します。-y フラグで質問をスキップしてデフォルト値を使用します。
3. 11tyのインストール
11tyをプロジェクトにインストール
npm install --save-dev @11ty/eleventy
4. 基本的なフォルダ構成の作成
my-website/ ├── src/ │ ├── _includes/ # テンプレート │ │ └── base.njk # 基本レイアウト │ ├── posts/ # 記事フォルダ │ │ └── 2024-01-01-first-post.md │ ├── index.md # トップページ │ └── about.md # 自己紹介ページ ├── _site/ # 生成されるサイト ├── .eleventy.js # 11ty設定ファイル └── package.json
5. 11ty設定ファイルの作成
.eleventy.js ファイル
module.exports = function(eleventyConfig) { // srcフォルダから_siteフォルダに出力 return { dir: { input: "src", output: "_site" } }; };
⚠️ 重要な注意点：
• src/ フォルダが作業場所です
• _site/ は自動生成されるため、直接編集しません
• _includes/ と _data/ は11ty特有の特殊フォルダです
3 基本的なページとテンプレートの作成
📄 基本レイアウトテンプレートの作成
src/_includes/base.njk ファイルを作成
基本レイアウトテンプレート
{{ title }} | My Website

トップページ
--- layout: base.njk title: ホーム --- # Welcome to My Website このWebサイトでは、技術やライフスタイルについて書いています。 ## 最新記事 - [初めての記事](/posts/2024-01-01-first-post/) お楽しみください！
src/about.md ファイルを作成（自己紹介ページ）
自己紹介ページ
--- layout: base.njk title: About --- # About Me はじめまして！私は... - Web開発に興味があります - 11tyを学習中です - Webサイトを通じて情報発信していきます よろしくお願いします！
src/posts/2024-01-01-first-post.md を作成（最初の記事）
最初のWebサイト記事
--- layout: base.njk title: 初めての記事 date: 2024-01-01 --- # 初めての記事 今日から11tyを使ってWebサイトを始めます！ ## 11tyについて 11tyは静的サイトジェネレーターです。 - マークダウンで記事が書ける - 高速なサイトが作れる - カスタマイズが自由 これからたくさん記事を書いていきたいと思います。
💡 ポイント：
• ファイル上部の --- で囲まれた部分は「フロントマター」と呼ばれる設定部分です
• layout でどのテンプレートを使うかを指定
• title はページタイトル
• date は記事の投稿日
4 サイトのビルドと確認
🚀 11tyでサイトを生成してみよう
1. package.jsonにスクリプト追加
package.jsonのscripts部分を編集
{ "name": "my-website", "version": "1.0.0", "scripts": { "build": "eleventy", "serve": "eleventy --serve", "start": "eleventy --serve" }, "devDependencies": { "@11ty/eleventy": "^2.0.1" } }
2. サイトをビルド（生成）する
サイト生成コマンド
npm run build

このコマンドを実行すると、_site フォルダに完成したWebサイトが生成されます。

3. ローカルサーバーで確認する
開発サーバー起動
npm run serve

ブラウザで http://localhost:8080 にアクセスして、サイトを確認できます。

💡 便利機能：
npm run serve を実行している間は、ファイルを編集すると自動でサイトが更新され、ブラウザも自動リロードされます！
⚠️ 重要： _site フォルダの中身は自動生成されるため、直接編集しないでください。編集は必ず src フォルダ内で行います。
5 GitHubでバージョン管理
🐙 GitHubアカウントとリポジトリの作成
1. GitHubアカウントを作成
  1. GitHubにアクセス
  2. "Sign up" をクリックして無料アカウントを作成
  3. メール認証を完了
2. 新しいリポジトリを作成
  1. GitHubにログイン後、右上の "+" → "New repository" をクリック
  2. リポジトリ名: my-website
  3. "Public" を選択
  4. "Create repository" をクリック
3. ローカルプロジェクトをGitで初期化
Gitリポジトリを初期化
git init
.gitignoreファイルを作成
echo "node_modules/ _site/ .DS_Store" > .gitignore
ファイルをステージング
git add .
初回コミット
git commit -m "Initial commit: 11ty website setup"
4. GitHubにプッシュ
リモートリポジトリを追加（ユーザー名を自分のものに変更）
git remote add origin https://github.com/YOUR_USERNAME/my-website.git
メインブランチ名を設定
git branch -M main
GitHubにプッシュ
git push -u origin main
💡 説明：
• .gitignore ファイルは、Gitで管理しないファイル・フォルダを指定します
• node_modules と _site は自動生成されるため、管理対象外にします
• GitHubにプッシュすることで、コードのバックアップと共有ができます
6 Netlifyで自動デプロイ
🌍 Netlifyでサイトを世界に公開
1. Netlifyアカウントを作成
  1. Netlifyにアクセス
  2. "Sign up" をクリック
  3. "GitHub" でサインアップを選択
  4. GitHubアカウントと連携
2. サイトをデプロイ
  1. Netlifyダッシュボードで "Add new site" → "Import an existing project"
  2. "Deploy with GitHub" を選択
  3. 先ほど作成した my-website リポジトリを選択
  4. 以下の設定を入力：

デプロイ設定

  • Build command: npm run build
  • Publish directory: _site
  • Base directory: (空白のまま)
3. デプロイ完了と確認
  1. "Deploy site" をクリック
  2. デプロイが完了すると、ランダムなURLが生成されます
  3. 例: https://amazing-curie-123456.netlify.app
  4. このURLをクリックして、サイトが公開されていることを確認
4. 独自ドメインの設定（オプション）
💡 独自ドメインを使いたい場合：
• Netlifyの "Domain settings" から独自ドメインを設定可能
• 無料プランでも独自ドメインは使用できます
• SSL証明書も自動で設定されます
⚠️ 自動デプロイについて：
今後GitHubにコードをプッシュすると、Netlifyが自動的にサイトを更新します。
記事を追加したり、デザインを変更したら、GitHubにプッシュするだけで本番サイトに反映されます！
7 Claudeを使った効率的な記事作成
🤖 AIで記事作成を効率化
1. Claude用のプロンプト例
記事作成プロンプト例
以下の条件でWebサイト用の記事を作成してください： タイトル: [記事のタイトル] 対象読者: [ターゲット層] 文字数: 約2000文字 形式: Markdown形式 含めて欲しい内容: - [ポイント1] - [ポイント2] - [ポイント3] フロントマターも含めて、11tyですぐに使える形式で出力してください。
2. 記事のファイル名規則

推奨ファイル名形式

  • YYYY-MM-DD-記事タイトル.md
  • 例: 2024-03-15-web-development-tips.md
  • 日本語は避け、英語とハイフンを使用
  • すべて小文字で統一
3. 記事投稿のワークフロー
  1. Claudeに記事作成を依頼
  2. 生成された内容を src/posts/ に保存
  3. npm run serve でローカル確認
  4. Gitにコミット・プッシュ
  5. Netlifyで自動的に本番サイトに反映
記事追加の基本コマンド
# 記事を追加した後 git add . git commit -m "Add new post: [記事タイトル]" git push
